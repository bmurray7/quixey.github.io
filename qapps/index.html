Quixey Content Sources

Quixey's search engine is powered by content from application sources using a variety of aquisition methods. The following list describes the current set of applications and their content sources used in both the Quixey SDK components the search API endpoint. As you go through the list, please note the following items as they will help with using the data provided:

  • Each row in the table or section with the app card represents a unique method of content acquistion for the specified Application. Multiple rows or sections with the sample Application name simply implies that there is a more than one provided mechanism to retrieve data and distinguished by one or more of the following: Entity Type, Geo-sensitivity, content aquisition frequency (TTL), and acqusition source (e.g. App, Web, Quixey Proprietary Cloud)
  • Entity Type - name given to the associated type and format (schema) of data retrieved. Please refer to the section in the documentation on DVC entity schemas for more details.
  • Vertical - name of the general category of search for which the content is associated. Please refer to the section in the documentation on DVC entity schemas for more details.
  • Geo - indicates whether this type of content is geo-sensitive or not. If yes, the content is aquired based on locality in relation to a specified lattitude/longitude pair. The supplied number indicates the accepted maximum precision. For instance 4 indicates the # of decimal places - 1.2345, 6.7890. Please refer to the DVC API Reference for more detail.
  • TTL - Time To Live or how long the data is cached by Quixey before being re-aquired. Value expressed in minutes. 0 = data is not cached but retrieved on demand.
  • Sources - location from which the data is acquired. When used with the Quixey API, it provides a platform filter from which to limit content searches. Please see the note in the DVC API Reference doc on allowable values.
  • Actions - set of commands provided by Quixey that allows the user to reach an application state or invoke an application function. (e.g. 'Watch' a movie on Hulu).
