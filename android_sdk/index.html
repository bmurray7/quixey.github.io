Quixey Mobile SDK for Android

Overview

The Quixey Mobile SDK for Android is a totally customizable SDK that is built for partners and third-party app developers and is fast and easy to integrate into device platforms and mobile apps. Quixey powers app search and discovery for major manufacturers, carriers, search engines, and web-sites across the globe. Quixey enables partners to create engaging user experiences that can be monetized successfully.

Highlights of the Quixey Mobile SDK for Android

This SDK provides access to the most relevant search and discovery results for contextual queries powered by Quixey's advanced Functional Search® through a search bar widget and a user-interest discovery widget. The following Quixey features are included:

  • Functional Search — A type of contextual user query that returns results for requested tasks and activities from apps and the app ecosystem.
    • Supports complex multi-word searches that describe functions the user intends to perform, for example “book a flight”, “buy movie tickets”, or “find food”.
    • Uses information such as location, time, user intent, and associated context to match and rank the search results from Quixey's rich deep view repository.
    • Returns both static and dynamic search results that can complete the task, deep linking directly into the functionality and content inside of apps.
    • Launches apps directly from the search bar with fallback to the mobile web or Google Play Store when the app is not already installed on the device.
  • Deep View Card (DVC) — An appealing concise expression of an app’s inner state, content, and functionality in a highly visual form.
    • Helps users to rapidly compare content and search results across multiple apps.
    • Supports stacks of cards organized into categories, subcategories, and personalized named favorites.
    • Provides customized Deep View Cards for a curated brand experience.
    • Associates Functional URLs to a deep state, deep linking to actions within an app.
  • Discovery — A display of featured and categorical collections or stacks of Deep View Cards.
    • Supports browsing to discover new app content based on user interest and context.
    • Curates and personalizes content, allowing users to save favorites and name new stacks.
    • Shares content and apps with friends and displays friends’ shared favorites.
    • Gives partners leverage to monetize their platforms.
    • Engages users with rich, curated brand experiences.

Build Variants

The Quixey Mobile SDK for Android supports the following build variants so that partners and third-party app developers can integrate Quixey’s features based on their business requirements.

Quixey Basic — Provides a search bar and a rendering view to render Deep View Cards based on a user-entered query. These are the most basic features of the SDK. Example code can be found in QxySample/sample-search.



Quixey Lite — Provides all the features of Quixey Basic while adding device search, supports a default view of user-interest Deep View Cards, and displays Now cards for calendar events, traffic updates, app usage, currency conversion, and other features. Example code can be found in QxySample/sample-home.



Quixey Full — Provides all the features of Quixey Lite while adding Deep View Card history, saving cards by user profile, and a tutorial. This is the largest build variant and provides the full power of Quixey Functional Search. Example code can be found in QxySample/sample-screens.



Prerequisites

Before starting integration, make sure you have been issued the following from Quixey:

  • Partner ID (partner_id) — unique partner API key for authentication.
  • Partner Secret (partner_secret) — partner secret key for authentication.
  • Optional App Default View — customization to use a partner-specific default view.

See the section on adding your partner configuration for more details on Quixey’s configuration options.

The following are prerequisites for integration:

  • Android 4.2 — API level 17 ( JELLY_BEAN_MR1 ) or later.
  • Process Architecture — ARM, Intel, and MIPS devices, each in 32-bit and 64-bit flavors.
  • Dependencies — see the section on Third-Party Libraries for a list of Android project dependencies.

Permissions

Quixey's SDK requires various Android permissions to operate.

The following normal permissions are used by all build variants:

  • ACCESS_NETWORK_STATE
  • ACCESS_WIFI_STATE
  • INTERNET
  • RECEIVE_BOOT_COMPLETED
  • VIBRATE
  • WAKE_LOCK

Depending on the build variant, the following additional Android OS permissions are required:

  • Calendar Group — READ_CALENDAR
  • Contacts Group — READ_CONTACTS, GET_ACCOUNTS
  • Location Group — ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION
  • SMS Group — READ_SMS
  • Storage Group — READ_EXTERNAL_STORAGE, WRITE_EXTERNAL_STORAGE (enforced at API level 19 or later)

Please note the following for each build variant:

  • Quixey Basic needs Location Group permissions.
  • Quixey Lite additionally needs PACKAGE_USAGE_STATS, Calendar Group, Contacts Group, and SMS Group permissions.
  • Quixey Full additionally needs Storage Group permissions.

Note: A Quixey home screen or launcher app may use system-level permissions if pre-installed on devices in the /system/app directory.

If you plan to use the Quixey Now cards feature, you will also need the permission android.permission.PACKAGE_USAGE_STATS, which is a system-level permission to provide access to device usage history and statistics. The user of the device can grant this permission through the Settings application. The manifest file of the Quixey Full sample QxySample/sample-screens/src/main/AndroidManifest.xml contains the following example of how this is declared:

<uses-permission android:name="android.permission.PACKAGE_USAGE_STATS"
        tools:ignore="ProtectedPermissions" />

Integration Steps

This section shows you how to integrate the Quixey Mobile SDK for Android into your project.
Follow these steps:

  1. Download the right Quixey libraries
  2. Add Quixey’s external dependencies
  3. Add Quixey’s partner configuration
  4. Update your application class
  5. Create your layouts
  6. Display your layouts
  7. Handle Quixey deep links
  8. Handle Google MAPS Android API integration
  9. Handle ProGuard obfuscation

STEP 1: Download the right Quixey libraries

Download the set of Quixey Android archive library .aar files that you need and add them to your project's lib folder. Also add these libraries to the Gradle dependency configurations file build.gradle. All three libraries are available for download from the Quixey Full sample libs directory: QxySample/sample-screens/libs.

Quixey Basic Library

  • Implement basic search functionality.
  • Use only sdk-standard-release.aar.

QxySample/sample-search/build.gradle:

compile(name: "sdk-standard-release", ext: "aar")

Quixey Lite Libraries

  • Implement search and discovery functionality without personalization.
  • Use both screens-lite-release.aar and sdk-standard-release.aar.

QxySample/sample-home/build.gradle:

compile(name: "sdk-standard-release", ext: "aar")
compile(name: "screens-lite-release", ext: "aar")

Quixey Full Libraries

  • Implement search, discovery, and personalization.
  • Use all three: discovery-release.aar, screens-lite-release.aar, and sdk-standard-release.aar.

QxySample/sample-screens/build.gradle:

compile(name: "sdk-standard-release", ext: "aar")
compile(name: "screens-lite-release", ext: "aar")
compile(name: "discovery-release", ext: "aar")

STEP 3: Add Quixey’s partner configuration

Before starting integration, make sure you have been issued product keys from Quixey.
Create a file named quixey_config.json in the assets folder. It should be at the same level as the java and res folders.

This configuration file (assets/quixey_config.json) must contain the following partner product keys and custom settings.

Product Keys

The product_keys object includes these string members:

  • partner_id — unique partner API key for authentication provided by Quixey.
  • partner_secret — partner secret key for authentication provided by Quixey.
  • partner_engagement — string to support internal development and testing.
    • The value should be "internaldev" to indicate the integration is in a staging environment.
    • When you are ready, this value should be changed to "release" to indicate the integration is in a production environment.

Custom Settings

The custom_settings object includes these Boolean and string members:

  • feature_screens — Boolean to enable/disable the Quixey Lite or Quixey Full features, including collections of Deep View Cards, and the ability to share these collections.
  • feature_save_deepview — Boolean to enable/disable user personalization by saving Deep View Cards for future use, only available in the Quixey Full feature set.
  • feature_autosuggest — Boolean to enable/disable auto-generated search term suggestions in the search bar.
  • feature_device_search — Boolean to enable/disable users to search their devices, including contacts, texts, and calendar entries.
  • preloaded_dv_url — optional base URL string of a deep view binding, preloaded to optimize performance. Specifies the location where remote artifacts are preloaded. Increases the build size, but reduces the download size.

Sample Partner Configuration File

QxySample/sample-screens/src/main/assets/quixey_config.json:

{
  "product_keys" : {
    "partner_id": "<partner-id>",
    "partner_secret": "<partner-secret>",
    "partner_engagement": "<partner-engagement>"
  },
  "custom_settings": {
    "feature_screens": true,
    "feature_save_deepview": true,
    "feature_autosuggest" : true,
    "feature_device_search" : true,
    "preloaded_dv_url": "file:///android_asset/deep_view/"
  }
}

STEP 4: Update your application class

Update your application class based on the feature set as follows:

  • For the Quixey Basic functionality you need to call the SDK initialization function QuixeySdk.initialize().
  • For the Quixey Lite feature set you need to extend your application class from the specified Quixey ScreensApplication class, which calls the initialization function.
  • For the Quixey Full feature set you need to extend your application class from the specified Quixey HomeApplication class, which calls the initialization function.

See the sample code below.

Quixey Basic Application Class

For Quixey Basic integration, initialize the SDK in your application class by calling the Quixey API QuixeySdk.initialize() and implementing a callback class, for example QuixeySdkCallback, that returns Quixey configuration values and handles Quixey events.

Sample Basic Application Code

QxySample/sample-search/src/main/java/com/quixey/sample/search/SampleApplication.java:

package com.quixey.sample.search;
import android.app.Application;
import com.quixey.android.QuixeySdk;
import java.util.Map;
public class SampleApplication extends Application {
    private boolean quixeyInitialized = false;
    public boolean isQuixeyInitialized() {
        return quixeyInitialized;
    }
    @Override
    public void onCreate() {
        super.onCreate();
        // initialize Quixey SDK
        QuixeySdk.initialize(this, new QuixeySdkCallback());
    }
    class QuixeySdkCallback implements QuixeySdk.Callback {
        @Override
        public void onNotSupportedSdkVersionCode(int sdkVersionCode) {
            // TODO: handle this event
        }
        @Override
        public void onRecommendedUpdateSdkVersionCode(int sdkVersionCode) {
            // TODO: handle this event
        }
        @Override
        public void onBadSystemPartnerKey() {
            // TODO: handle this event
        }
        @Override
        public String getConfigJsonPath() {
            return "quixey_config.json";
        }
        @Override
        public void onInitComplete() {
            //Quixey is initialized
            //Use this to show progress bar or hide/unhide Quixey View
            quixeyInitialized = true;
        }
        @Override
        public Map<String, String> getAltPartnerCredentials() {
            return null;    //Partner Config is already part of quixey_config.json
        }
    }
}

Quixey Lite & Quixey Full Displays

For Quixey Lite and Quixey Full displays, use an Activity or Fragment to display the layout and to pass events to the Quixey view. Quixey Lite requires that user interest categories of Deep View Cards be pre-loaded (see the ArrayList interest in the sample code below). The Quixey Full feature set provides an additional user interface flow that allows end-users to select and save their interests (and the ArrayList interest can be removed).

Examples of common user interest categories include:

  • news
  • local
  • politics
  • tech news
  • movie trailers
Sample Lite Main Activity Code

package com.quixey.sample.home;
import android.app.Activity;
import android.app.AlertDialog;
import android.content.DialogInterface;
import android.content.Intent;
import android.os.AsyncTask;
import android.os.Bundle;
import android.view.View;
import android.view.Window;
import com.quixey.screens.library.assist.IQuixeyView;
import java.util.ArrayList;
public class MainActivity extends Activity {
    private static final String LOG_TAG = MainActivity.class.getSimpleName();
    private IQuixeyView mQuixeyScreen;
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        getWindow().requestFeature(Window.FEATURE_NO_TITLE);
        setContentView(R.layout.activity_main);
        mQuixeyScreen = (IQuixeyView) findViewById(R.id.quixey_screen);
        mQuixeyScreen.onCreate(savedInstanceState);
        ArrayList<String> interest = new ArrayList<String>();
        interest.add("NEWS");
        interest.add("LOCAL");
        interest.add("MOVIE TRAILERS");
        interest.add("POLITICS");
        interest.add("TECH NEWS");
        mQuixeyScreen.setInterest(interest);
        new Loading().execute();
    }
    class Loading extends AsyncTask<Void, Void, Boolean> {
        long startTime = System.currentTimeMillis();
        long TIMEOUT = 30 * 1000;
        @Override
        protected Boolean doInBackground(Void... params) {
            while (System.currentTimeMillis() - startTime < TIMEOUT) {
                if (((SampleApplication)getApplication()).isSdkInitialized()) {
                    return Boolean.TRUE;
                }
                try {
                    Thread.sleep(1000);
                } catch(Exception e) {}
            }
            return Boolean.FALSE;
        }
        @Override
        protected void onPostExecute(Boolean result) {
            if (result) {
                findViewById(R.id.progress).setVisibility(View.GONE);
                mQuixeyScreen.setVisibility(View.VISIBLE);
            } else {
                findViewById(R.id.progress).setVisibility(View.GONE);
                AlertDialog alertDialog = new AlertDialog.Builder(MainActivity.this).create();
                alertDialog.setTitle(getString(R.string.error_title));
                alertDialog.setMessage(getString(R.string.error_body));
                alertDialog.setButton(AlertDialog.BUTTON_NEUTRAL, getString(R.string.error_exit),
                        new DialogInterface.OnClickListener() {
                            public void onClick(DialogInterface dialog, int which) {
                                dialog.dismiss();
                                finish();
                            }
                        });
                alertDialog.show();
            }
        }
    }
    @Override
    public void onBackPressed() {
        if (mQuixeyScreen != null && !mQuixeyScreen.onBackPressed()) {
            super.onBackPressed();
            //Handle Application Specific implementation
        }
    }
    @Override
    protected void onNewIntent(Intent intent) {
        super.onNewIntent(intent);
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onNewIntent(intent);
        }
        //Handle Application Specific implementation
    }
    @Override
    protected void onPause() {
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onPause();
        }
        //Handle Application Specific implementation
        super.onPause();
    }
    @Override
    protected void onResume() {
        super.onResume();
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onResume();
        }
        //Handle Application Specific implementation
    }
    @Override
    protected void onDestroy() {
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onDestroy();
        }
        //Handle Application Specific implementation
        super.onDestroy();
    }
    @Override
    protected void onStart() {
        super.onStart();
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onStart();
            mQuixeyScreen.onSpotLight(true);
        }
        //Handle Application Specific implementation
    }
    @Override
    protected void onStop() {
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onStop();
            mQuixeyScreen.onSpotLight(false);
        }
        //Handle Application Specific implementation
        super.onStop();
    }
    @Override
    protected void onActivityResult(int requestCode, int resultCode, Intent data) {
        boolean handled = false;
        if (mQuixeyScreen != null) {
            handled = mQuixeyScreen.onActivityResult(requestCode, resultCode, data);
        }
        if (!handled) {
            super.onActivityResult(requestCode, resultCode, data);
            //Handle Application Specific implementation
        }
    }
    @Override
    public void onRequestPermissionsResult(int requestCode, String[] permissions, int[] grantResults) {
        if (mQuixeyScreen != null) {
            mQuixeyScreen.onRequestPermissionsResult(requestCode, permissions, grantResults);
        }
        //Handle Application Specific implementation
    }
}

STEP 8: Enable Google MAPS Android API integration

If your app uses Quixey Now cards, the Quixey Now traffic feature uses the Google MAPS Android API. To use the Google Maps Android API, you must register your app project on the Google Developers Console and get a Google API key. This API key is based on a short form of your app’s digital certificate, known as its SHA-1 fingerprint. Quixey must add your app’s package name and SHA-1 fingerprint from the release certificate keystore file used to sign your application to the Google API Console for billing purposes, otherwise the traffic Now card will not work.

For more details see Google Maps Android API documentation

STEP 9: Enable ProGuard obfuscation

The Quixey Android archive library .aar files are already obfuscated, if you are obfuscating your Android application, add the line below in your ProGuard Rules so that the Quixey .aar files are not obfuscated more than once.

QxySample/sample-screens/proguard-rules.pro:

Quixey Discovery

Bundled within Quixey Lite and Quixey Full are capabilities that allow you to view and manipulate collections of Deep View Cards. These collections are referred to as Quixey Screens. A 'Golden State Warriors' Screen, for example, might contain one or Deep View Cards relating to news and information about the namesake's basketball team. A 'Tech Startup' Screen might contain Deep View Cards related to early-stage tech companies, their founders and venture funding. Content for screens can be any combination of app or web results.

Screens are grouped by Category and any Screen can belong to one or more Categories. In the example above, the Golden State Warriors Screen can be categorized under both News AND Sports. Categories are heirarchical and can be nested to any level (e.g. Sports -> Basketball -> NBA).

Quixey supports two types of Screens:

  • Quixey-Screens — a pre-build set of custom Screens developed by Quixey for general use or as a service for partners
  • User-Screens — a set of Screens created by the end-user based on a selection of specific Deep View Cards from Search Results or from other Screens

Screens API

Using Quixey Screens within your app is easy. Quixey provides the following set of classes and methods for working with Screens and their associated Categories.

ScreensClient Class

// The helper class to retrieve and render predefined Quixey screens.
public class ScreensClient {
    // Instantiate the class with a context like "Activity"
    public ScreensClient(Context context);
    // Retrieve all the supported interests.
    public void getSupportedInterests(Callback<List<String>, GatewayError> callback);
    // Set (save) the user interests.
    public void setUserInterests(Set<String> interests);
    // Get the user interests.
    public Set<String> getUserInterests();
    // Launch the Home screen that consists of deep view cards relevant to the user interests.
    public void launchHomeScreen();
    // Launch the Local screen.
    public void launchLocalScreen();
    // Launch the specified screen.
    public void launchScreen(String id);
    // Retrieve the specified category - a container of subcategories and screens.
    // Specify an empty "categoryUrl" to retrieve the topmost (root) category.
    public void getScreensCategory(String categoryUrl, Callback<ScreensResult, GatewayError> callback);
}

ScreensResult Class

// Result of a screen category retrieval.
public class ScreensResult {
    // Get the subcategories.
    public List<CategoryEntry> getChildren();
    // Get all the screen entries contained in the subcategories.
    public List<ScreensEntry> getScreens();
    // Get the url of the targeted category.
    public String getCategoryUrl();
}

CategoryEntry Class

// A category entry.
public class CategoryEntry {
    // Get the category url.
    public String getCategoryUrl();
    // Get all the screens contained in the category.
    public List<String> getScreenUrls();
    // Get the category's display name.
    public String getDisplayName();
    // Get the category's remote icon.
    public String getDisplayIcon();
}

ScreensEntry Class

// A screen entry
public class ScreensEntry {
    // Get the screen url.
    public String getScreenUrl();
    // Get the screen id.
    public String getScreenId();
    // Get the screen's display name.
    public String getDisplayName();
    // Get the screen's remote icon.
    public String getDisplayIcon();
}

Quixey Widgets

Quixey Widgets represent discrete functional components that can be used to enhance your application Quixey search, app actions and content discovery. As mentioned in the layout creation integration steps above, Quixey widgets are made available through the supplied library build variants. The following describe the list of available widgets.

Search Widget

Provides components for creating a basic search experience including a Search Bar and a Search Results view.

Name: com.quixey.android.ui.custom.widget.DeepViewSerp
Library: sdk-standard-release.aar
Example:

<com.quixey.android.ui.custom.widget.DeepViewSerp
    android:id="@+id/search_serp"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    app:includeSearchBar="true"
    app:serpSoln="@string/solution_tag"
    android:visibility="gone"/>

Options:

  • includeSearchBar — set to true to show a search bar control, false to use just the results view
  • serpSoln — used for partner analytics. Specifies a unique string to represent the widget placement in the user interface and to distinguish it from other instances of the widget in the app.

Quixey Lite View Widget

Provides all the features of the Search Widget while adding device search, supports a default view of user-interest Deep View Cards, and displays Now cards for calendar events, traffic updates, app usage, currency conversion, and other features.

Name: com.quixey.screens.library.ui.custom.widget.QuixeyScreenLite
Library: screens-lite-release.aar
Example:

<com.quixey.screens.library.ui.custom.widget.QuixeyScreenLite
    android:id="@+id/quixey_screen"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:background="@color/screen_app_background"
    app:searchBar="true"
    app:nowCards="true"
    app:titleImage="@drawable/ic_splash_logo_bigger"
    android:visibility="gone"
    />

Options:

  • searchBar — set to true to show a search bar control, false to use just the results view
  • nowCards — set to true to show a search bar control, false to use just the results view
  • titleImage — used to set a drawable title.

Quixey Full Widget

Provides all the features of the Quixey Lite View Widget while adding Deep View Card history, saving cards by user profile, and a brief tutorial.

Name: com.quixey.screens.library.ui.custom.widget.QuixeyScreen
Library: discovery-release.aar
Example:

<com.quixey.screens.library.ui.custom.widget.QuixeyScreen
    android:id="@+id/quixey_screen"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:background="@color/screen_app_background"
    android:visibility="gone"
    />

Quixey Actions Widget

Provides available actions for the apps installed on the device. Actions can be as simple as toggling Android settings or complex such as opening your favorite finance app directly into your checking account. The widget is implemented as an Android Cardview component.

Name: com.quixey.android.ui.deepview.container.view.ActionCard
Library: screens-lite-release.aar, discovery-release.aar
Example:

<com.quixey.android.ui.deepview.container.view.ActionCard
    android:id="@+id/action_card"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:actionType="all"
    />

Options:

  • actionType — sets the type of actions displayed in the card. Allowable values include:
    • "device" — only show device actions
    • "app" — only show app actions
    • "all" — show both device AND app actions
