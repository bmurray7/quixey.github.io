Product Overview

Introduction

Quixey Search is a mobile product for retrieving content quickly interacting with apps. Users can interact with the product with explicit queries describing specific needs as well as implicit needs based on user context. Many mobile needs enable users to perform actions in the physical world. The product helps users accomplish these action-based needs more quickly and easily.

Product Features

Search

A user is able to invoke the search feature-set from any state in the Quixey app. Search input consists of two possible options: entering a query using the keyboard, or using the device’s microphone to speak a search query.

Device Search

As the user enters a query, content available on the device such as installed apps, events and contacts are matched to the user’s query and rendered virtually instantly. Mobile users no longer have expectations of where content and information is physically stored. Some needs, such as “open a particular app” or “call a particular person” can be fulfilled immediately by Device Search, and furthers the goal of minimizing the time required to complete an action.

Autosuggest

If no on-device content is relevant, the user will see autosuggest results which allows the user to save time typing by selecting recent queries or likely complete queries based on the user’s partial query. Autosuggest is improved by using query logs, knowledge, and data feedback to predict a user’s information need without requiring the user to type a textual query completely. This is especially valuable on mobile, where typing is often slower than on desktop devices.

Deep Search / App Search

If a user selects an autosuggest result, or if the user submits the query by tapping enter, the user will see a single SERP returning Deep View Cards for both App Search and Deep Search results. These DVCs are served by our core DVC Search online service, consisting primarily of the following stages:

  • Understanding the user’s query
  • Retrieving either particular pieces of content or predicting * particular parts of apps relevant to the user’s possible needs
  • Ranking these results
    Rendering the best possible Deep View Card for each result

Universal Search

Mobile users rarely consider whether or not the content or information is located physically on the mobile device or in the cloud. The popularity of music streaming services and other cloud-based applications proves that abstracting the data layer from the users’ concerns delivers utility.

Universal Search is our key offering which mixes and ranks content from Deep, App and Device search components; ranking DVCs by relevance, agnostic to which search component retrieved the result, reflects this new reality of mobile computing and matches users’ expectations.

Richer Deep View Cards

Deep View Cards deliver value to the user by presenting the most fresh and rich content related to an information need to help the user consider a result and how an App can fulfill a given need. Most importantly, a user can tap on a DVC and navigate to the most relevant part of the corresponding app.

A user will often see DVCs that are dynamic and feature real-time content extracted directly from the app, web or an API. For example, if a user enters a query such as “gs stock price”, “weather” or “taxi”, the DVC is rendered with up to the second freshness sourced directly from the app.

Contextual Discovery

A first-time user is onboarded through a simple flow which allows the user to configure several topics which are of interest to the them.

For existing users or after the onboarding process a user will see their home screen featuring DVCs that are relevant to the user based on trending DVCs for their location. DVCs will be ranked by relevancy based on preferences, previous activities and selected topics.

In addition to the home screen, a user can navigate to other screens with trending, popular or informational DVCs related to topics likely to be relevant to the user’s interests and needs.

Actions

A user can speak or type a query such as “play music” which implies a need that can be formalized as “open my favorite music app, and play my most recent or favorite music”. Quixey is able to understand this need, and return a DVC allowing the user to perform this action.

Actions usually involve not only navigating to a deep link, but also interacting with an app in a subtle or unique way. Quixey has a technical solution for this problem, and when a user taps on an action DVC in Quixey, Quixey is able to perform the action magically.

  • Personal — Many actions involve interacting with apps which result in real-life results, such as hailing a cab or ordering food, and other actions involve sensitive personal data. Quixey is able to handle needs such as “open portfolio” and “order thai food” locally on the device, and navigate the user to potentially personal information without ever compromising privacy or security.
  • Bread Crumb Trail — Quixey’s SDK features proprietary Bread-crumb Trail (BCT) technology that is able to navigate users to any state with an app, regardless of the existence of deep links. This significantly expands the coverage of DVC navigation relative to other deep linking solutions, and links the user’s directly to the most relevant action, saving users time to perform actions (TTA).
