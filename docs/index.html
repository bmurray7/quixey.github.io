Introduction

Quixey Search is a mobile product for retrieving content quickly and interacting with apps. Users can interact with the product by using explicit queries to describe specific needs as well as through content tailored to their specific interests. The benefit to the end user is the ability to satisfy action-based needs through apps on their mobile device with minimum time and effort (i.e. fewest clicks)

Product Features

Our Search online service is a complex series of systems that work together to deliver the best possible results to the user. In short Quixey's systems are responsible for:

  • Understanding the user’s query
  • Retrieving either particular pieces of content or predicting particular parts of apps relevant to the user’s possible needs
  • Ranking these results
  • Rendering the best possible Deep View Card for each result

The core of Quixey's Mobile Search product offering is described in the following set of features.

Device Search

As the user enters a query, content available on the device such as installed apps, events and contacts are matched to the user’s query and rendered virtually instantly. Mobile users no longer have expectations of where content and information is physically stored. Some needs, such as “open a particular app” or “call a particular person” can be fulfilled immediately by Device Search, and furthers the goal of minimizing the time required to complete an action.

Vertical Search

Quixey also provides users the ability to focus their search queries by specifying a vertical or scope. These verticals are specific categorizations of content that provide further context to Quixey's search system thereby ensuring the user is provided a consistent highly relevant set of results.

Autosuggest

If no on-device content is relevant, the user will see autosuggest results which allows the user to save time typing by selecting recent queries or likely complete queries based on the user’s partial query. Autosuggest is improved by using query logs, knowledge, and data feedback to predict a user’s information need without requiring the user to type a textual query completely. This is especially valuable on mobile, where typing is often slower than on desktop devices.

Deep Search / App Search

If a user selects an autosuggest result, or submits a query through an input method, the user will be presented with a single SERP returning a set of richly-formatted cards for both App Search and Deep Search results. These Deep View Cards (DVCs) are served by our Search online service and presented via our Android SDK

These Deep View Cards deliver value to the user by presenting fresh and rich content related to an informational need along with one or more paths to meet this or other related needs. At the very least, Quixey provides the user the ability tap on a DVC and navigate to the most relevant part of the corresponding app.

A user will often see DVCs that are dynamic and feature real-time content extracted directly from the app, web or an API. For example, if a user enters a query such as “gs stock price”, “weather” or “taxi”, the DVC is rendered with up to the second freshness sourced directly from the app.

Universal Search

Mobile users rarely consider whether or not the content or information is located physically on the mobile device or in the cloud. For example, the popularity of music streaming services and other cloud-based applications proves that abstracting the data from users’ concerns delivers utility.

Universal Search is our key offering which mixes and ranks content from Deep, App and Device search components; ranking DVCs by relevance, agnostic to which search component retrieved the result, reflects this new reality of mobile computing and matches users’ expectations.

Contextual Discovery

A first-time user is on-boarded through a simple flow which allows the user to configure several topics which are of interest to the them.

For existing users or after the on-boarding process a user will see their home screen featuring DVCs that are relevant to the user based on trending DVCs for their location. DVCs will be ranked by relevancy based on preferences, previous activities and selected topics.

In addition to the home screen, a user can navigate to other screens with trending, popular or informational DVCs related to topics likely to be relevant to the user’s interests and needs.

Actions

Along with simple navigational queries, a Quixey user can speak or type a query such as “play music” which implies a need that can be formalized as “open my favorite music app, and play my most recent or favorite music”. Quixey is able to understand this need, and return a DVC allowing the user to perform this action.

Actions represent the set of operations a user can perform for the entities contained within an app. These actions can range from very simple operations such as "Call this Restaurant" or "View this hotel in Maps" to more complex operations such as "Order this item" or "Check my account balance". Actions can also be associated with device functions allowing the user to perform such operations as "Turn on the flashlight" or "Toggle Wifi".

Personal Actions

Many actions involve interacting with apps that result in practical operations that often use sensitive personal information such as hailing a cab, ordering food, or checking account balances. Quixey is able to handle these needs locally on the device, and navigate the user with potentially personal information without ever compromising privacy or security.

Product Comparison

Quixey's Search technology is made available to partners through a set of Android native libraries (SDK) or via a set of public APIs. While both offer core search capabilities, the Android libraries provide the best overall integration experience by taking advantage of Quixey's on-device presence enabling such features as Device Search, BCT-powered actions and Contextual Discovery. Deep Linking Support

Quixey Deep Linking is a key product feature that provides seamless integration between a user intention and the set of information and capabilities supplied by applications on the user's device. However, not all apps are built to expose their content and functionality through external means and so Quixey has provided partners with support for a number of Deep Linking methods including the following:

  • URIs - these are industry-standard ways to invoke applications that allow access to content and/or functionality.
  • Intents (Android) - allows you to start an activity in another app by describing a simple action you'd like to perform (such as "view a map" or "take a picture"). Quixey maintains a catalog of apps with intent support and uses these whenever possible to perform app actions.
  • Universal Links (iOS) - allows users to tap a link to an app's website and get seamlessly redirected to it's installed app without going through Safari. If the app isn’t installed, tapping a link to the website opens the website in Safari.
  • BCTs - Quixey’s SDK features proprietary Bread-crumb Trail (BCT) technology that is able to navigate users to any state with an app, regardless of the existence of deep links. This significantly expands the coverage of DVC navigation relative to other deep linking solutions, and ensures user intentions are carried out reliably and predictably by apps resulting in shortest time to action (TTA).
