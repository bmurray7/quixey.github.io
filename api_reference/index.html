Quixey API Reference

Overview

Quixey's API is a set of endpoints that allow you to retrieve information about apps and app content. {more here}

Some use cases for Quixey API include the following:

  • Mobile app developers - provide your users with advanced search capabilities that includes app and app content results.
  • Social Marketing Partner - search for and generate content for Ad units in various formats including Twitter Ad Cards, Facebook Open Graph, and Schema.org.

Authentication

Authentication credentials are required for all requests made through Quixey's API endpoints. Credentials consist of a Quixey-generated partner ID and secret key. Do not share your secret API keys in publicly accessible areas such GitHub, client-side code, and so forth.

Versioning

All API calls should contain a version string contained withing the base portion of the URL. For example:

https://apiserver.quixey.com/ext/v1.1/dvcs

Errors

All failure conditions are returned in the response as an object contiaining a message, HTTP status and Quixey return code.

Name Description
msg String describing the nature of the error
status HTTP status code (see list for details)
rc Return Code. Possible values include:

Example error response

{
  "msg": "Public access to Quixey's API has been disabled. You must provide a partner ID. Please contact partners@quixey.com for a verified partner account.",
  "status": 401,
  "rc": 10
 }

HTTP Status Code Summary

The table below lists the most commone HTTP Status Codes returned by Quixey's API servers.

Code Meaning
200 OK -- Everything worked as expected.
400 Bad Request -- The request was unacceptable, often due to missing a required parameter.
401 Unauthorized -- No valid API key provided.
402 Request Failed -- The parameters were valid but the request failed.
404 Not Found -- The requested resource doesn't exist.
409 Conflict -- The request conflicts with another request (perhaps due to using the same idempotent key).
429 Too Many Requests -- Too many requests hit the API too quickly. We recommend an exponential backoff of your requests.
500 Internal Server Error - Something went wrong on Quixey's end.
502 Bad Gateway -- One of Quixey's upstream servers received an invalid response
503 Service Unavailable -- The server is currently unable to handle the request due to a temporary overloading or maintenance of the server.
504 Gateway Timeout -- The server, while acting as a gateway or proxy, did not receive a timely response from the upstream server specified by the URI

Endpoints

/search

Returns a set of Quixey Deep View Card results in a variety of formats.

Request Parameters

Name Type Validated Description
partner_id string Y [Required] Quixey-generated partner ID
partner_secret string Y [Required] Quixey-generated partner key
q string Y [Required] the full query text. Ex: q=thai
geo string Y [Required] The geographic location of the user in latitude and longitude using decimal degrees with negative numbers for South and West; an example location in Palo Alto, CA is geo=37.422054,-122.136725
apps json list Y List of apps for which search results will be limited. Ex: apps=[askme.com,snapdeal.com,expedia.com]
types json list N List of types for which search results will be limited. Specific types can be obtained through the /apps API endpoint response. Ex: types=[Event]
limit int Y The maximum number of Deep View Cards to return; up to a maximum of 20. Default = 10.
dv_limit int N The maximum number of Deep Views to return for each Deep View Card. Default = 5.
locale string Y Sets the locale for a search. When available, descriptions, prices, and download links will be localized to the given locale.
Ex: locale=en_US
Notes:
  • China partners are limited to zh_CN
  • US partners are limited to en_US, en_SG, and en_GB.
fix_spelling boolean Y Used to correct spelling in the q parameter. Default=true
platform json list N A JSON-encoded array of Quixey store IDs. If this field is present, only return Deep View Cards with editions that are available on one or more of the specified stores
Notes:
  • The store names are not validated.
  • Supported stores are 'itunes' (iOS) and 'googleplay' (Android).
  • Upper and lowercase do not effect the results

Ex: platform=[googleplay,yunos]
format string Y How to format the response output. Allowable values are:
  • twitter.ad_card
  • quixey
  • open_graph
mode json list Y Specifies the type of search to perform and the results that appear in the response object.
Allowable values are:
  • app = app search results
  • deep = deep search results
  • app_deep = both app and deep search results

Default=deep

Response Structure

The /search response is primarily an ordered set of Deep View Cards (DVCs) from one or more apps. The first DVC in the response is highest ranked by Quixey should suffice for most use cases. This number is controlled by the ‘limit’ request parameter.

Root

Field Type Present Description
deepViewCards DeepViewCard[] Y An unordered set of Deep View Cards
searchId string Y Quixey-assigned search ID for this search request
sessionId string Y Quixey-assigned session ID for this search request

DeepViewCard - a grouping of similar search results or Deep Views from a particular app. Contains the following fields:

Field Type Present Description
deepViews DeepView[] Y A set of Deep Views
@id FURL Y A Quixey-assigned identifier for the card. This value should be unique among all cards in the response
cardName string N A human readable name for card
appName string N A human readable name for the App that provided content for the card
ttl long Y Time-to-live. The average freshness in seconds for the Quixey function used to generate the card. Default 86,400s or 1 day.

DeepView - set of data describing the details for a particular entity within an app. Contains the following fields.

Field Type Present Description
format.name Format Y The deep view content formatted by Format.
@id FURL Y A Quixey Functional URL of the deep state
url URL Y The best web URL for the deep state. If not available, should navigate to the app store.
geo Geolocation N The geolocation of the object.

Geolocation

Field Type Present Description
latitude Double Y Latitude of location.
longitude Double Y Longitude of location.
distanceMeters Double Y Distance in meters.
distanceMiles Double Y Distance in miles.

debug

Field Type Present Description
warnings string[] Y warning logs
errors string[] Y error logs
exceptions string[] Y exception logs
info string[] Y info logs

Format Types

twitter.ad_card

Field Type Present Description
@type string Y Currently this has default value as "image_app_download"
name string Y It is deepviewcontent's display content's name/product_name.
description string N Not supported in v1.1.
image_url URL N It is deepviewcontent's display content's image/product_image.
app_cta string Y Currently this has default value as "INSTALL_OPEN"
iphone_deep_link URL N It is deep view's stateAccessInfo's accessurl for edition having platform = ios
googleplay_deep_link URL N It is deep view's stateAccessInfo's accessurl for edition having platform = android

open_graph

Field Type Present Description
og:title string Y It is deepviewcontent's display content's name.
og:image string N It is deepviewcontent's display content's image.
og:description string N It is deepviewcontent's display content's description.
og:url string N It is deepviewcontent's display content's url.
og:site_name string Y this is appName.
og:type string Y "website" constant value for this format
